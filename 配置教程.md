# 配置与快速上手教程

本教程旨在帮助用户完成系统功能的初始化配置。关于环境安装与启动，请参阅 [README.md]。

---

## 1. 系统配置（API Key 池）

入口：`系统配置` -> `API Key 池`

本系统支持多模型轮询。在使用生成功能或高级抓取前，必须配置相应的 API Key

- 至少一个大模型（建议Deepseek）
- 至少一个抓取（建议Firecrawl）

### 操作步骤
- **新增 Key**: 点击“新增 Key”，选择对应的 Provider（如 `deepseek`, `openai`, `firecrawl` 等）。
- **Provider 说明**:
  - `deepseek/openai`: 用于内容生成与 AI 辅助编辑。
  - `firecrawl`: 用于高级网页搜索与抓取。
  - `aliyun_iqs`: 用于阿里云集成搜索。
- **状态管理**: 可以在列表中随时启用或禁用某个 Key。

---

## 2. 数据采集功能

### 2.1 数据源管理
入口：`数据采集` -> `数据源管理`

在这里定义您的信息抓取入口。

- **创建数据源**: 支持 `网页 URL`、`API`、`n8n Webhook（暂未完善）` 等类型。
- **触发抓取**: 在列表的操作栏点击“立即触发抓取”图标。系统将立即启动异步任务对目标内容进行采集、抽取和清洗。
- **CSS 选择器**: 对于网页数据源，如果默认抓取不理想，可以配置 CSS 选择器来精确指定正文区域。

### 3.2 抓取记录
入口：`数据采集` -> `抓取记录`

查看所有已完成的抓取任务及其结果。

- **查看详情**: 点击“查看”，可以查看系统抽取出的结构化正文、清洗统计数据以及原始 HTML 预览。
- **快捷抓取**: 点击页面右上角的“快捷抓取”按钮，可以输入一个临时 URL 进行一次性抓取测试，无需创建正式数据源。
- **预览模式**: 快捷抓取支持“预览”功能，可以在保存入库前先查看抽取效果。

---

## 3. 常见问题
- **抓取失败**: 检查目标网站是否有反爬限制，或 API Key 池中的 `firecrawl` Key 是否配置正确。
- **内容为空**: 尝试在数据源配置中增加特定的 CSS 选择器以定位正文内容。
